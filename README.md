# cyprob-plugin-source

This repository is the source-of-truth for Cyprob plugin source code.

## Purpose

- Store Cyprob-authored plugins as first-class source artifacts.
- Provide a clean source layer for YAML, WASM, and future plugin formats.
- Receive normalized upstream content through a separate import path, not directly from distribution feeds.
- Feed the content factory, which builds publishable plugin artifacts for plugin feeds.

## Scope

This repository is for source content only.

It should contain:

- Plugin manifests and source files
- WASM plugin source projects and build metadata
- Cyprob-authored detection logic
- Normalized upstream-derived plugin source after review/import

It should not contain:

- Built feed shards
- Published `index.json` files
- CDN/R2 deployment artifacts
- Factory output

## Planned Role In The Pipeline

```text
upstream import(s) -> cyprob-plugin-source -> cyprob-content-factory -> cyprob-plugin-feed
```

## Content Model

The exact directory layout will be finalized later, but this repository is intended to hold:

- community-compatible source
- enterprise-only source
- alpha/stable promotion candidates at the source level

Distribution channel separation remains a feed concern, not a source repo concern.

## Rules

- Manual edits are allowed here.
- This repo is the editable plugin source layer.
- Distribution artifacts must be generated by factory, not committed here.
- Nuclei or other upstream imports must arrive through a controlled normalization path.
